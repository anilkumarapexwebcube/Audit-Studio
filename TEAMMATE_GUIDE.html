1. Install the extension (one time)

1

Get the folder

Your admin will share a folder called gsc-audit-studio via Google Drive or Slack. Download it and unzip it.

CRITICAL: Use the exact path below. Do NOT extract anywhere else. The folder path is used by Chrome to generate the extension's identity. If everyone on the team uses the same path, everyone gets the same identity, and the tool works for everyone. If you use a different path, your sign-in will fail with a "redirect_uri_mismatch" error.
Exact path to extract to:
Windows: C:\Tools\gsc-audit-studio
Mac: /Users/Shared/gsc-audit-studio

How to do this on Windows:

  1. Open File Explorer, navigate to C:\
  2. If a folder called Tools does not exist, create it: right-click in empty space → New → Folder → name it Tools
  3. Move the downloaded zip file into C:\Tools
  4. Right-click the zip → Extract All → in the destination box, make sure it says C:\Tools\gsc-audit-studio (NOT C:\Tools\gsc-audit-studio\gsc-audit-studio)
  5. Click Extract
  6. Verify: open C:\Tools\gsc-audit-studio in File Explorer. You should see manifest.json directly inside (not in a sub-folder)
Do NOT put this folder on Desktop, Downloads, Documents, OneDrive, or any other location. Only C:\Tools\gsc-audit-studio. If you must change the path, message your admin first so they can authorise your specific install.
2

Open Chrome's extensions page

Type chrome://extensions in the address bar and press Enter.

3

Enable Developer mode

Toggle the Developer mode switch at the top right corner of the extensions page. This shows extra buttons including "Load unpacked".

4

Load the extension

Click the Load unpacked button at the top left. A folder picker opens. Navigate to your gsc-audit-studio folder and click Select Folder.

If you see an error "Manifest file is missing or unreadable", you probably selected a parent folder by mistake. Make sure you select the folder that has manifest.json directly inside it, not a folder that contains another folder.
5

Pin the extension to your toolbar

Click the puzzle piece icon at the top right of Chrome. Find GSC Audit Studio in the list and click the pin icon next to it. The extension icon now appears in your toolbar permanently.

2. Connect your Google accounts

Your admin will tell you which GSC Google accounts you are responsible for (for example searchconsoleanalyticsaccess8@gmail.com or analytic.w3era@gmail.com). You sign into each one once on your machine. After that, the extension uses them silently for months without asking.

1

Open the dashboard

Click the GSC Audit Studio icon in your Chrome toolbar. The dashboard opens in a new tab.

2

Go to the Accounts tab

Click Accounts in the navigation.

3

Connect each account

For each Google account you need to use:

  1. Type the email in the box (e.g. searchconsoleanalyticsaccess8@gmail.com)
  2. Click Connect Account
  3. A Google sign-in window pops up
  4. Sign in with that account's password
  5. When you see "GSC Audit Studio wants to access your Google Account", click Continue or Allow
  6. The window closes and the account appears in the table with a green Connected badge

Repeat for every account on your list. Connect all of them in one session so you don't have to revisit later.

What you just did: Each connection stores a refresh token locally on your machine. The extension uses this token to call the GSC API silently in the background, no further sign-in needed. Tokens last for months (rarely revoked by Google) and are tied to this specific Chrome profile on this specific machine.

3. Generate your first report

1

Click the extension icon

Dashboard opens. You land on the Run Audit tab.

2

Type or paste a domain

In the text box, type a domain (e.g. azskymechanical.com) on a single line. No https:// needed, no trailing slash, just the bare domain.

3

Check the preview

Below the text box, the preview shows the account email that will be used and the connection status. If you see a green dot, you are ready. See the preview legend below for what each colour means.

4

Click Generate Report(s)

The log panel below will show progress: account being used, property URL, API data being fetched, screenshots being captured, PPTX being built. Total time is usually 10 to 20 seconds.

5

Find your PPTX file

Open your Downloads folder. You'll see a file named like GSC_Audit_azskymechanical_com_2026-06-09.pptx. This is your report, ready to send to the client.

4. Generate multiple reports at once

Same flow, just paste multiple domains in the text box, one per line:

azskymechanical.com
subtelink.com
buzzkill.com.au
challengeot.com
brand-gaming.com

Click Generate. The extension processes each domain in sequence. For 5 domains, expect 1 to 2 minutes total. Walk away and come back, all 5 PPTX files will be in your Downloads folder.

Different accounts in the same batch is fine. If domain 1 maps to account A and domain 2 maps to account B, the extension switches between them automatically (as long as both are connected on your machine).

5. Understanding the preview indicators

When you type domains in Run Audit, each line shows a coloured dot indicating its status:

IndicatorMeaningWhat you do
Green Domain is in the master sheet, the assigned Google account is connected on this machine Nothing. Just click Generate.
Yellow Domain is in the master sheet, but the assigned account is not yet connected on this machine Click Generate anyway. The extension will pop up a sign-in window for that account once. After signing in, processing continues automatically. Future reports for that account never ask again.
Red Domain is NOT in the master sheet Contact your admin and ask them to add the domain to the master sheet (or run "Sync domains from GSC" if the account already has access).

6. Troubleshooting

What you seeWhat it meansWhat to do
"Not in master sheet" for a domain you expect to work Domain has not been added to GSC_Config yet Send the domain name to your admin. They will add it. Then click Refresh in the extension (or just retry, the extension auto-fetches on each Generate).
"Session expired" or "Please reconnect the account" Google revoked the refresh token (rare, usually after very long inactivity) Go to Accounts tab. Click Disconnect on that account. Re-enter the email and click Connect Account again. Takes 30 seconds.
Manual Action or Security Issues slide is blank in the PPTX The screenshot capture timed out before the GSC page finished loading Try again. If it happens repeatedly, ask your admin to increase the wait time.
"No property found for X on this account" The mapping says account A owns the domain, but the GSC API says account A has no such property. Probably an old mapping. Tell your admin. They will re-sync the master sheet from GSC.
OAuth window says "This app is blocked" or "Access blocked" Your personal Google account (the one signing in) is not in the OAuth test users list Send your admin the exact email address you tried to sign in with. They will add it to the test users list. Wait 5 minutes and try again.
OAuth window says "Error 400: redirect_uri_mismatch" Your extension folder is in the wrong path on disk, so Chrome generated a different extension ID than what the admin authorised Close the error. Open chrome://extensions, remove the extension. Move your folder to exactly C:\Tools\gsc-audit-studio (Windows) or /Users/Shared/gsc-audit-studio (Mac). Load unpacked again. Try sign-in again.
Anything else Could be anything Screenshot the log panel in Run Audit tab AND any error popup. Send both to your admin via Slack.

7. Things to avoid

  • Do not delete the extension folder while the extension is loaded in Chrome. If you must move it, remove the extension first via chrome://extensions, move the folder, then Load unpacked from the new location.
  • Do not share the extension folder with anyone outside the team. The folder contains OAuth credentials that grant API access.
  • Do not sign out of the GSC Google accounts in your Chrome browser unless your admin asks you to. Signing out invalidates the screenshot capture for Manual Action and Security Issues slides.
  • Do not modify any files in the extension folder. If a feature is broken or missing, tell your admin.
  • Do not paste OAuth credentials (Client ID, Client Secret), Apps Script URLs, or master sheet URLs in any Slack channel, email, or chat outside the team. These are sensitive.

Need help?

Contact your admin with:

  • A screenshot of the log panel in the Run Audit tab
  • A screenshot of any error popup
